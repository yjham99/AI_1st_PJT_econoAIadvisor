## 3. 안티그래비티 주입용 설정 스크립트 (Coding & Config)

안티그래비티의 설정 파일(`mcp_config.json` 혹은 `rules.md`)에 적용할 가이드라인입니다.

### [Step 1] 지능형 게이트웨이 설정 (Logical Routing)

참모진이 자신의 업무를 수행할 때 호출할 API의 우선순위를 강제합니다.

```markdown
# Model Routing Rules
IF role == "Echo" THEN USE "gemini-3-flash"
IF role == "Tech" THEN USE "gpt-oss-120b"
IF role == "Biz" THEN USE "claude-4.5-sonnet"
IF role == "Cabin" THEN USE "claude-4.5-thinking"

```

### [Step 2] 토큰 효율화 SOP (Standard Operating Procedure)

안티그래비티의 `agent_business_manual.md`에 추가할 내용입니다.

1. **Context Caching:** 반복되는 기업 배경 정보와 참모진 규칙은 반드시 캐시를 활용하여 중복 과금을 방지한다.
2. **Summary Pipeline:** 에코(Flash)가 수집한 모든 원문 데이터는 상급 모델로 전달되기 전 반드시 1차 요약을 거친다.
3. **Threshold:** 단일 작업에 50k 이상의 토큰이 예상될 경우 사장님의 승인을 요청한다.

## 4. 안티그래비티 대상 실행 명령 (Prompt Injection)

이제 안티그래비티에게 아래 내용을 복사해서 던지시면 됩니다. 시스템이 이 구조를 스스로 이해하고 재세팅할 것입니다.

> **"안티그래비티, 새로운 [모델 운용 효율화 계획]을 하달한다.**
> 1. **모델 맵핑:** 앞으로 '에코'는 `gemini 3 flash`를 기본 엔진으로 사용하고, '비즈'와 '캐빈'은 `claude 4.5` 시리즈를 사용해.
> 2. **비용 최적화 루틴:** 대량의 데이터를 읽을 때는 무조건 `flash` 모델로 먼저 스캔하고, 유의미한 정보만 추려서 고사양 모델에 전달하는 '계층적 분석' 프로세스를 표준으로 정한다.
> 3. **반영 확인:** 위 내용대로 내 참모진들의 업무 규칙(`agent_business_manual.md`)을 업데이트하고, 앞으로의 모든 보고서에서 사용된 모델과 예상 토큰 효율을 하단에 표기해."
